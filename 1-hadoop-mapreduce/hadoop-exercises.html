Exercise 1: Inverted Index

Write a MapReduce program in Java that will construct an "inverted index" for a number of input files.

In computer science, an inverted index (also referred to as a postings file or inverted file) is a database index storing a mapping from content, such as words or numbers, to its locations in a table, or in a document or a set of documents (named in contrast to a forward index, which maps from documents to content). Ref: Wikipedia

The purpose of an inverted index is to allow fast full text searches.

In this exercise, you are going to create a very simple inverted index.

Example

Suppose we have the following two files:

file1.txt

This is a test
This is so much fun

and file2.txt

This is another file
I have no more inspiration

Then the result should be the following

a       file1.txt:1
another file2.txt:1
file    file2.txt:1
fun     file1.txt:1
have    file2.txt:1
i       file2.txt:1
inspiration     file2.txt:1
is      file2.txt:1;file1.txt:2
more    file2.txt:1
much    file1.txt:1
no      file2.txt:1
so      file1.txt:1
test    file1.txt:1
this    file1.txt:2;file2.txt:1

Thus, for each word in file1.txt and file2.txt (converted to lower case) we know which file(s) it appeared in and how often it appeared in that file.

Possible Strategy

  • Write a Mapper that produces (word@fileName, 1) pairs for each word in the file.
  • Write a Combiner that combines (word@fileName, <1, 1, 1, .... 1>) pairs into (word@fileName, n)
  • Write a custum Partitioner that ensures that all words go to the same reducer. See here for an explanation about the method that you need to implement. You can mimick this implementation.
  • Write a Reducer that produces the desired output. The code for the Reducer is not so straightforward. Remember: the input key will be word@fileName but we can only produce a new output when we have seen all the files for the current word. The MapReduce framework will ensure that the reducer sees the keys in sorted order. Thus, the reducer will need an instance variable that you can use to keep track of the previous word and an instance variable to gather all the files (and the counts) that go with that word. You can only emit something to the context when you notice a word different from the previous word, because then you know that you have seen all the keys containing this particular word. However, this will be problematic for the last word. Why? Hence, you should emit the last word in the cleanup method of the Reducer, see this link.

Some Tips

  • Start from the WordCount program to get the boilerplate code.
  • Calling context.getInputSplit() is a good first step to determine the name of the file that is currently being processed.
  • The output types of the Mapper and Combiner must be the same.

Exercise 2: Matrix Multiplication

In this exercise you will implement a MapReduce algorithm to multiply two sparse matrices.

Matrix Multiplication

Matrix multiplication is an operation whereby an m×cm \times cm×c matrix MMM and a c×nc \times nc×n matrix NNN are combined into a m×nm \times nm×n matrix PPP.

The formula for the (i,j)(i,j)(i,j)-entry of the matrix PPP is

Pi,j=∑k=1cMi,kNk,jP_{i, j} = \sum_{k=1}^c M_{i, k} N_{k, j} Pi,j​=k=1∑c​Mi,k​Nk,j​

E.g. let MMM and NNN be given by

M=[142506]andN=[1−103]M = \begin{bmatrix} 1 & 4 \\ 2 & 5 \\ 0 & 6 \end{bmatrix} \quad\text{and}\quad N = \begin{bmatrix} 1 & -1 \\ 0 & 3 \end{bmatrix} M=⎣⎡​120​456​⎦⎤​andN=[10​−13​]

then

P=[111213018.]P = \begin{bmatrix} 1 & 11 \\ 2 & 13 \\ 0 & 18. \end{bmatrix} P=⎣⎡​120​111318.​⎦⎤​

In some applications we may need to multiply matrices that are too big to be stored in the main memory of a computer. Moreover, we assume that the matrices are sparse matrices, i.e. matrices that contain a lot of zeros.

Storing Matrices

Assume that matrices are stored in the format

MatrixName,row_index,column_index,value

where only non-zero values are stored.

In this format, the matrices MMM and NNN above are stored as

M,0,0,1
M,1,0,2
M,0,1,4
M,1,1,5
M,2,1,6

and

N,0,0,1
N,0,1,-1
N,1,1,3

In this format, we assume that the matrices are sparse, i.e. that many entries of the matrix are zero. These files could be stored in HDFS.

Map Reduce algorithm (in two steps)

Notice how in the formula for matrix multiplication we need to multiply elements from column kkk in MMM with elements from row kkk in NNN. It thus makes sense to use kkk as the key for first mapper, so that these entries will end up at the same reducer.

The first mapper will take an entry

M,i,k,m_ik

and produce a key-value pair

(k, (M, i, m_ik))

for the first matrix. For the second matrix the input entry is

N,k,j,n_kj

and the produced key-value pair is

(k, (N, j, n_kj))

The first reducer then takes these entries (all with the same value for k) and produces key-value pairs of the following form

( (i,j), m_ik * n_kj)

The second mapper acts as the identity function: it takes

((i,j), m_ik * n_kj)

as the value and produces a record with (i,j) as the key and m_ik * n_kj as the value.

The second reducer then takes (i,j) as the key and the corresponding multiplied values m_ik * n_kj as the values and simply sums these values.

The resulting record should be of the same format as the input files.

Implementing the algorithm

Implement the above algorithm in Hadoop MapReduce.

Take the following into account:

  • You will need to write two mappers and two reducers. Use a command line argument to decide which part of the algorithm to execute
    MatrixMultiply <stage> <input-dir> <output-dir>
    so the first time you execute
    MatrixMultiply 1 matrix-input matrix-tmp
    and the second time you execute
    MatrixMultiply 2 matrix-tmp matrix-output
    so the output of the first stage is the input to the second stage
  • In the first reducer you may need to loop over the Iterable twice. This does not work, so you will need to cache all these values locally, e.g. using a Java List. Note: this assumes that the number of values stored will not exhaust the main memory (RAM) of the computer running this reduce job. This should/could be fine as the maximum number of values is at most the number of columns/rows in the first/second matrix. We assume that the matrices are sparse so in reality the number of values will be much lower than this maximum value.
  • Even though the second mapper is supposed to be the identity function, you will still need some work because the default key is different from the logical key produced by the first reducer.
  • Make sure the final output of your algorithm matches the input. To this end, the second reducer should not use a key.
  • You can pass the names of the matrices in the Configuration object, e.g.
Configuration conf = new Configuration();
conf.set("FirstMatrixName", "M");

Exercise 3: Matrix Multiplication in One Step

Matrix multiplication can be done with a single map and single reduce, but then the map and reduce functions have to do a bit more work.

Strategy:

  • In the map function each entry M,i,k,m_ik gives rise to multiply key-value pairs. All these key-value pairs have keys of the form (i,j) where j takes on values from 0 to the number fo columns of NNN minus 1. The value in each case is (M,k,m_ik). For an entry N,k,j,n_kj the keys are of the form (i,j) where i takes on values from 0 to the number of rows of MMM minus 1. The value here is (N,k,n_kj)
  • In the reduce step we get keys of the form (i,j). We will need to match the values (M,k,m_ik) with the values (N,k,n_kj) in order to multiply m_ik and n_kj. Finally, these products need to be summed together in order to produce the output.
